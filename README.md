# Buddy

> A Netflix-style app for AI-generated book (soon: movie) summaries.

Dark-mode catalog of books grouped into genre rows, with a featured "hero" book on
top. Tap a card and a floating modal streams a ~2-minute summary word-by-word, like a
chat. The catalog and summaries are generated by Google Gemini; book covers come from
the free Open Library API.

The app calls the free Google Gemini API. You need a key:

1. Create a free key at [aistudio.google.com/apikey](https://aistudio.google.com/apikey).
2. **Local dev:** copy `.env.example` to `.env.local` and paste your key:
   ```bash
   cp .env.example .env.local
   # then edit .env.local: REACT_APP_GEMINI_API_KEY=...
   ```
3. **Deploys (GitHub Pages):** add a repo secret named `REACT_APP_GEMINI_API_KEY`
   (GitHub â†’ repo Settings â†’ Secrets and variables â†’ Actions â†’ New repository secret).
   Both deploy workflows inject it at build time.

> âš ď¸Ź GitHub Pages is static hosting, so the key is bundled into the published
> JavaScript and is therefore visible to anyone. Treat it as disposable and rotate it
> if it gets abused. To hide it, front the API with a small serverless proxy later and
> swap the implementation inside `src/services/gemini.ts`.

The catalog is generated once per day and cached in the browser (`localStorage`);
summaries are cached for a week. So normal browsing uses very little of the free quota.
